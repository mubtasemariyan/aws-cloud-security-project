# AWS Cloud Security Project

## Overview
This project demonstrates how to configure and secure cloud infrastructure using AWS security services and best practices.

---

## 🎯 Why I Built This
Cloud security is a critical skill in modern cybersecurity roles. This project helped me understand identity management, logging, and secure cloud configurations.

---

## 🛠️ Tools Used
- AWS Free Tier
- IAM (Identity and Access Management)
- S3 (Simple Storage Service)
- CloudTrail
- GuardDuty (optional)

---

## ⚙️ What I Did
- Created secure IAM users with least privilege access
- Configured a private S3 bucket with restricted permissions
- Enabled CloudTrail logging for monitoring activity
- Set up basic cloud security monitoring

---

## 📊 Results / Metrics
- Configured 5+ IAM policies with least privilege principles
- Secured S3 bucket preventing public access
- Logged and monitored 100+ AWS events via CloudTrail

---

## 🧠 What I Learned
- Cloud identity and access management
- Importance of least privilege security model
- How cloud logging and monitoring works
- Basics of AWS security architecture

---

## 📌 Key Takeaway
This project demonstrated my ability to secure cloud infrastructure and apply real-world AWS security practices.
